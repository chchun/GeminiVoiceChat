# 프로젝트 요구사항 정의서 (PRD) - v1.1

## 1. 개요
Gemini Live 스타일의 실시간 대화형 AI 안드로이드 앱 클라이언트 개발. 초기 버전은 가짜(Mock) 데이터를 활용해 네트워크 단절 상태에서도 완벽한 UI/UX 인터랙션을 보장하는 것을 목표로 함.

## 2. 데이터 모델 명세 (Data Models)
Claude는 아래 구조를 기반으로 데이터 레이어를 설계해야 한다.

### 2.1 ChatMessage
- `id`: String (UUIDv4 자동 생성)
- `text`: String (화면에 표시될 메시지 내용)
- `senderType`: SenderType (USER, AI)
- `timestamp`: Long (System.currentTimeMillis())
- `isStreaming`: Boolean (AI 답변이 진행 중인지 여부, USER는 항상 false)

---

## 3. 화면 상태 및 UI/UX 구성 (Jetpack Compose)

화면은 크게 **[텍스트 채팅 모드]**와 **[보이스 모드 오버레이]** 두 가지 상태를 가진다.

### 3.1 텍스트 채팅 모드 (기본 메인 화면)
- **대화 이력창 (LazyColumn):**
    - USER 메시지: 우측 정렬, 보라색 말풍선, 백색 글씨.
    - AI 메시지: 좌측 정렬, 회색 말풍선, 흑색 글씨.
    - **스마트 스크롤:** 새로운 메시지가 추가되거나 AI 답변이 스트리밍될 때, 사용자가 과거 내역을 위로 올려보고 있는 상태(스크롤 업 상태)가 아니라면 자동으로 최하단으로 스크롤 포커스를 맞춰야 한다.
- **하단 입력 바:**
    - `TextField`: 한 줄 입력창, 텍스트 입력 시 전송 버튼 활성화. 비어있을 때는 전송 버튼 비활성화.
    - `마이크 버튼`: TextField 왼쪽에 배치. 클릭 시 **보이스 모드(3.2)**로 전환.
    - `전송 버튼`: 클릭 시 텍스트 전송 로직 실행 및 입력창 초기화.

### 3.2 보이스 모드 (Gemini Live 스타일 오버레이)
텍스트 화면 위에 전체 화면(혹시 하단 시트)으로 반투명 오버레이가 등장한다.
- **상태 변화에 따른 애니메이션 효과 구현 필 수:**
    - `LISTENING (유저가 말하는 중)`: 화면 중앙에 마이크 파형 애니메이션(Waveform)이 유저 목소리 크기에 따라 부드럽게 움직임.
    - `THINKING (AI가 생각 중)`: 파형이 잔잔한 로딩 루프 애니메이션으로 변경됨.
    - `SPEAKING (AI가 말하는 중)`: AI 목소리 출력에 맞춰 파형 애니메이션이 다시 활성화됨.
- **하단 컨트롤러:**
    - `종료(X) 버튼`: 보이스 모드를 즉시 종료하고 메인 텍스트 채팅창으로 돌아감. (대화 내용은 유지)

## 4. 핵심 기능 흐름 및 로컬 비즈니스 로직

### 4.1 텍스트 스트리밍 흐름
1. 유저가 텍스트 입력 후 전송 클릭.
2. `ChatMessage(USER)`가 리스트에 즉시 추가됨.
3. UI는 'AI 생각 중(Loading)' 상태를 표현 (예: 말풍선 안에 세 개의 점이 깜빡이는 애니메이션).
4. `MockAiRepository`로부터 답변을 `Flow<String>` 형태로 스트리밍 처리.
5. 리스트의 마지막 AI 메시지 객체의 `text` 필드에 글자가 실시간으로 누적되며 타자 치듯 화면에 갱신됨. (`isStreaming = true`)
6. 스트리밍이 완료되면 `isStreaming = false`로 전환.

### 4.2 로컬 음성 인식 및 합성 (STT & TTS) 파이프라인
*현재 단계에서는 구글 클라우드를 연동하지 않고, 안드로이드 OS 내장 기능을 사용함.*

- **STT (Speech To Text) 구현 가이드:**
    - 안드로이드 내장 `SpeechRecognizer`와 `RecognizerIntent.ACTION_RECOGNIZE_SPEECH` 사용.
    - 보이스 모드 진입 즉시 실행되며, 오디오 권한(`Manifest.permission.RECORD_AUDIO`)이 없을 경우 Compose 런타임에서 권한 요청 팝업을 띄워야 함. 거부 시 보이스 모드 종료 및 토스트 메시지 출력.
    - 유저가 말을 멈추면(End of Utterance) 자동으로 인식을 종료하고, 획득한 텍스트를 AI Repository로 전달.
- **TTS (Text To Speech) 구현 가이드:**
    - 안드로이드 내장 `TextToSpeech` 클래스 활용.
    - AI 스트리밍 답변이 완료되거나, 혹은 문장 단위(`. `, `? `)로 청크가 완성될 때마다 해당 텍스트를 즉시 TTS 음성으로 출력해야 함.
    - 보이스 모드가 종료되거나 유저가 중간에 말을 시작하면 현재 출력 중인 TTS 음성을 즉시 `stop()` 해야 함.

## 5. 비기능적 요구사항 및 예외 처리
- **네트워크 강인성:** 실제 서버 연동을 고려하여 비즈니스 로직은 비동기 코루틴(`Dispatchers.IO`)에서 실행되어야 하며, UI 스레드를 차단해서는 안 됨.
- **메모리 관리:** 오디오 녹음 및 TTS 인스턴스는 화면 소멸(onDestroy 또는 Composable dispose) 시점에 명확하게 `release()` 및 `shutdown()` 되어 메모리 누수를 방지해야 함.
